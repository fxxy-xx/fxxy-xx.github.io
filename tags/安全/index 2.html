  • SSRF-服务端请求伪造

    2019-09-26 |
    • { 安全 }
    • SSRF：服务端请求伪造漏洞
    • 原理：由攻击者构造的攻击链接传给服务端执行造成的漏洞，一般用在外网探测或攻击内网服务。漏洞大多数发生在连接外网与内网的跳板机中
    • 成因：服务器未对发起请求的URL进行有效检测和过滤。
    • 哪些地方容易出现SSRF：
      • 网站的分享功能
      • 在线翻译，通过URL地址进行翻译
      • PC到手机的转码服务（通过url把原网址内容转成适合手机阅读的情况）
      • 图片文章的加载、下载、收藏
